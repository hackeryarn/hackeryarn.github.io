Model methods

When we write model methods, or any methods for that matter, note that we do not add a type to self. We do, however, add a return type to all methods. This isn't much of an advantage for functions like __str__ but becomes helpful with was_published_recently.

was_published_recently also depends on time and a database call. This means that the function can fail and is not mathematically pure. Because of this we use the impure_safe decorator.

This decorator will change the type from bool to IOResultE[bool, Exception]. This additional wrapper means that we can't use the value without first unwrapping it and handling the exception. Later, we will see how to handle all this elegantly.

Also, notice that we did not wrap __str__ despite it being an impure function that touches the database. Since we declared it as a magic method, Python won't know how to handle our nice wrapper.

from django.db import models
from django.utils import timezone
from returns.io import impure_safe
from returns.unsafe import unsafe_perform_io


class Question(models.Model):
    ...

    @impure_safe
    def _str(self) -> str:
        return self.question_text

    def __str__(self) -> str:
        value = self.safe_str().value_or("No question_text found")
        return unsafe_perform_io(value)

    def was_published_recently(self) -> bool:
